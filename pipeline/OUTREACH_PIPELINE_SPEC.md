# webmorp.art — автоматизированный outreach pipeline (спецификация)

Целевой процесс: найти медленные/устаревшие сайты → cold email → при ответе «yes» запустить AI-pipeline → preview → оплата → выдача файлов.

**Связь с кодом в репозитории:** сейчас реализованы поиск лидов через DuckDuckGo (`finder.py`), SQLite (`leads_db.py`), экспорт черновиков и отправка через SMTP/Graph (`export_outreach_md.py`, `send_outreach.py`). Описанное ниже — целевая архитектура и порядок внедрения; части Phase 1–4 ещё предстоит автоматизировать или подключить внешние сервисы.

---

## Фаза 1 — Поиск лидов

### Источник A: Google PageSpeed Insights API

- **В репозитории:** скрипт `psi_enrich.py` + модуль `pagespeed.py` — для строк в `leads` с email записывают `psi_mobile_score` (0–100, mobile Lighthouse) и `psi_checked_at`. Ключ: `PAGESPEED_API_KEY` или `GOOGLE_API_KEY` в `pipeline/.env`.
- Вход: список URL из БД после finder (или из CSV в будущем).
- Вызов **PageSpeed Insights API** (квоты — в [документации Google](https://developers.google.com/speed/docs/insights/v5/get-started)).
- **Фильтр «медленные»:** при `--psi-slow-only` в экспорте/рассылке берутся только лиды с **измеренным** score **&lt; 55** (порог настраивается); быстрые сайты отсекаются.

### Источник B: BuiltWith

- Фильтры (пример): технология **jQuery 1.x** или **Bootstrap 3** или **WordPress 4–5**; страна **US / UK / AU**; сегмент **small business**.
- Экспорт в **CSV**; на бесплатном плане типично ограничение порядка **~200 записей** (актуальные лимиты — в BuiltWith).

### Квалификация перед отправкой

| Критерий | Действие |
|----------|----------|
| PageSpeed mobile **< 55** (или эквивалент из API) | Берём |
| Сайт открывается, бизнес выглядит активным (например свежие отзывы в **Google Business Profile**) | Берём |
| На сайте есть **прямой email** (не только форма контакта) | Берём |
| Уже **Squarespace / Wix / Webflow** (по сигнатурам в HTML/хосте) | Пропускаем |
| Крупная сеть / франшиза | Пропускаем |

**Дополнение к текущему finder:** отсев **не-статических** визиток (`_looks_like_non_static_brochure`): магазины, CMS, конструкторы, типовые SPA/SSR; плюс эвристики «устаревшести»; PageSpeed и BuiltWith — отдельные источники для `leads` (`psi_mobile_score`, `psi_checked_at`, …).

## Фаза 2 — Cold email

### Тема

- Варианты: `your site` или `[domain] — quick question`
- **Избегать в теме:** `free`, `redesign`, `no cost` и похожие формулировки (частые спам-триггеры).

### Тело письма (версия A — рекомендуется)

```
Hi [First name],

[domain] scores [X]/100 on mobile speed — that's costing you leads.

We'll rebuild it free. You see it before you pay. Keep it → $100.

Reply "yes" to see your new site.

Pavel · webmorp.art
```

### Персонализация

- `[domain]` — реальный домен.
- `[X]` — реальный балл mobile из PageSpeed (или округлённый из API).
- `[First name]` — из локальной части email, CRM или GMB; если нет — **«Hi there»**.

### Follow-up последовательность

| День | Смысл |
|------|--------|
| 0 | Первое письмо |
| 3 | Скриншот/пример похожего сайта из ниши — «what we did for a similar business» |
| 7 | «Taking on 3 projects this week…» — мягкий дефицит |
| 14 | «Closing your file — reply anytime» |

### Инфраструктура рассылки

- **Instantly.ai** или **Lemlist** (мульти-триггерные цепочки, прогрев доменов).
- Ориентир: **до 50 новых контактов в день** с одного домена; для масштаба — несколько доменов (например 3×50 ≈ 150/день).
- **Прогрев** почтовых ящиков/домена **2–3 недели** до агрессивного объёма.

**Текущий репозиторий:** `send_outreach.py` шлёт «вручную» из очереди БД без автоматических follow-up; интеграция с Instantly/Lemlist — отдельный этап (экспорт CSV / API / Zapier).

## Фаза 3 — AI pipeline (триггер: ответ «yes»)

**Ядро принятия решений по вёрстке и структуре:** модель **Gemini** из Google AI Studio (в спецификации указано «3.1 Pro» — при внедрении подставить **актуальный идентификатор модели** из Studio, например семейство 2.5 / следующий GA).

### Шаг 1 — Скрейп сайта

- **Playwright** или **Puppeteer** (Node.js): полный HTML (или упрощённый DOM), **inner text**, `title`, meta description, `h1`, все `img` + `alt`, телефон, email, сохранённый **PageSpeed score** (если уже считали).

**Текущий репозиторий:** `scraper.py` — облегчённый HTTP+BeautifulSoup; для «полного» скрейпа под pipeline целесообразен отдельный Node/Playwright сервис или расширение пайплайна.

### Шаг 2 — Анализ и контент (Gemini)

Промпт возвращает **строгий JSON** с полями:

| Поле | Описание |
|------|-----------|
| `hero_headline` | Новый H1, max **8 слов**, benefit-first |
| `hero_subheadline` | Одно предложение, конкретный результат |
| `services` | Массив: `name`, `description` (~20 слов), `icon_keyword` |
| `why_us` | 3 пункта доверия: `title` (~4 слова), `body` (~15 слов) |
| `cta_primary`, `cta_secondary` | Тексты кнопок |
| `seo_title` | До **60** символов |
| `seo_description` | До **155** символов |
| `color_scheme` | Одно из: `professional-blue`, `warm-orange`, `natural-green`, `bold-red`, `elegant-dark` |
| `template` | Одно из: `salient`, `radiant`, `spotlight`, `studio` |
| `image_prompts` | Ровно **3** промпта для генерации изображений |

### Шаг 3 — Изображения

- **Imagen 3** (или актуальный image API в экосистеме Google / Vertex) по промптам из JSON.
- Стоимость в спецификации: ориентир **~$0.12/сайт** (пересчитывать по прайсу Google).

### Шаг 4 — Сборка HTML

- Шаблоны **Tailwind Plus** (лицензия допускает клиентские проекты — хранить копии шаблонов вне публичного репо, если лицензия того требует).
- Каждый шаблон заранее размечен плейсхолдерами `{{...}}`; скрипт подставляет JSON + пути к сгенерированным изображениям.

**Текущий репозиторий:** `designer.py` — зачаток генерации под другой поток; под эту спецификацию нужен отдельный **builder** модуль.

### Шаг 5 — Preview deploy

- **Netlify CLI** или Netlify API: деплой на субдомен вида `preview-abc123.netlify.app`.
- Только для превью клиенту; не production домен заказчика.
- Free tier: ориентир **до ~100 деплоев/мес** (проверять актуальные лимиты Netlify).

### Шаг 6 — Screenshot

- **Puppeteer** или **ScreenshotOne** API — full-page скрин для вложения/ссылки в письме.

### Шаг 7 — Письмо клиенту

```
Here's your redesign: [preview URL]

To keep it: $100 → [gumroad link]
To deploy on your domain: +$100 more

Link expires in 7 days.
```

## Фаза 4 — Оплата и доставка

### Gumroad

1. Продукт **«Your webmorp site»** — **$100**. Digital product; в описании явно: **no refunds after download**.
2. Продукт **«Deploy to your domain»** — **$100**, отдельная покупка.

### Webhook

- Gumroad шлёт webhook на событие **`sale`** (email покупателя, product id).
- По email находится запись в БД лида/заказа; обновляется **download URL** на ZIP конкретного сайта; Gumroad может уведомить покупателя об обновлении выдачи (настроить по документации Gumroad).

### Содержимое ZIP

- HTML, CSS, изображения, **README** (деплой на Netlify / Vercel / GitHub Pages).

## Порядок запуска (рекомендуемый)

1. **10 лидов вручную** → pipeline вручную → письма вручную → оценить качество AI и отклик **до** тяжёлой автоматизации.
2. После первых результатов — автоматизировать **scrape → AI → build → deploy → email**.
3. После стабильной конверсии — автоматизировать **проспектинг + рассылку** (Instantly и т.д.).
4. Последним шагом — **Gumroad webhook** и автоматическая выдача ZIP.

## Риски и контроль

| Риск | Митигация |
|------|-----------|
| Спам-фильтры | Прогрев 2–3 недели; ≤50 писем/день с домена; чистая тема без триггер-слов |
| Плохой вывод AI | **Обязательный human review ~5 мин** на сайт перед отправкой preview клиенту |
| Чарджбэк после скачивания | Явно в Gumroad: no refunds на digital после загрузки |
| Бесконечные правки | В письме и на Gumroad: **1 итерация включена**, далее **$25** за раунд |

## Human review (обязательный шаг)

Перед тем как отправлять клиенту ссылку на preview:

- Проверить бренд, факты, контакты, опечатки, битые картинки, мобильную вёрстку.
- Не полагаться на полностью автоматическую отправку без просмотра до фазы зрелости процесса.

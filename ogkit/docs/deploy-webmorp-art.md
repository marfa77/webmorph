# Почему `webmorph.art/ogkit` → 404 на GitHub Pages

Сайт **[webmorph](https://github.com/marfa77/webmorph)** сейчас, скорее всего, отдаётся через **GitHub Pages**. Pages публикует **только статические файлы** из ветки (например `index.html`, папки с `.html`).

Путь **`/ogkit`** на таком хостинге означает: «в репозитории должен быть файл  
`ogkit/index.html` (или аналог) в **той же ветке**, что публикуется Pages».  
Next.js-приложение OGKit туда **само не попадает**: это отдельный билд (Node, API routes, `/api/og/*` и т.д.).

Отсюда стандартная страница **404 — File not found** от GitHub Pages.

---

## Что делать (нормальный вариант)

**Деплой OGKit — на Vercel** (или другой хостинг для Next.js), **не** на GitHub Pages для этого приложения.

1. В Vercel: **New Project** → импорт **`marfa77/webmorph`** (или отдельный репо, если вынесете `ogkit`), **Root Directory** = `ogkit`, если приложение лежит в подпапке.
2. Env: как в `README` / `.env.example` — в т.ч.  
   `NEXT_PUBLIC_BASE_PATH=/ogkit`,  
   `NEXT_PUBLIC_APP_URL=https://www.webmorp.art/ogkit`
3. **Домен:** либо подключить **`www.webmorp.art`** к проекту Vercel с OGKit (если весь сайт там), либо вынести приложение на сабдомен, например **`ogkit.webmorp.art`**, и сделать редирект/ссылку с лендинга.

Пока DNS / CNAME для `www.webmorp.art` указывают **только на GitHub Pages**, запросы к `/ogkit` обрабатывает **только** Pages — и ищет там несуществующий статический файл.

---

## Если корень сайта остаётся на GitHub Pages

Нужно явно решить маршрутизацию:

- **Вариант A:** в DNS отдельно направить поддомен или путь на Vercel (зависит от регистратора и от того, где «сидит» основной домен).
- **Вариант B:** перенести **весь** публичный фронт на Vercel (лендинг + ogkit в одном или двух проектах с rewrites).

Временный костыль (не рекомендуется как финал): в репо для Pages положить **`ogkit/index.html`** с редиректом на реальный URL Vercel — только чтобы не видеть 404, не заменяет нормальный деплой Next.

---

## Кратко

| Хостинг            | Подходит для OGKit (Next + API)? |
|--------------------|-----------------------------------|
| GitHub Pages       | Нет (только статика; нет вашего Next-билда в `/ogkit` без костылей) |
| Vercel             | Да                                |

После переноса OGKit на Vercel и правильных **env + домена** URL вида **`https://www.webmorp.art/ogkit`** будет открывать уже собранное приложение, а не поиск `index.html` на Pages.
